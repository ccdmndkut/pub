4. Container Queries

Apply styles based on the size of a parent container, not just the viewport.

This feature allows components to adapt to their *container's* size. Full implementation across browsers is evolving. Check caniuse.com for current support.

6. CSS Variables (Custom Properties)

Define reusable values for consistent styling and easier theming (like the light/dark mode toggle you see!). This text uses a variable for its color.

7. Enhanced Viewport Units

Use `svh`/`svw` (Small), `lvh`/`lvw` (Large), and `dvh`/`dvw` (Dynamic) units for more precise sizing relative to the viewport, accounting for browser UI elements.

50vh x 50vw Box

8. Cascade Layers (@layer)

Gain explicit control over the CSS cascade to manage styles from different sources (frameworks, themes, components) more predictably.

This text's final style (red/orange) is determined by layer order, not specificity alone.

9. Content Visibility

Improve rendering performance by telling the browser to skip rendering for off-screen content (`content-visibility: auto`).

Potentially Deferred Section

If this section were large and off-screen, `content-visibility: auto` could speed up initial page load by deferring its rendering until it's about to be scrolled into view.

The browser intelligently manages the rendering of content marked with `content-visibility: auto`.

Specify a specific region of an image or video to display, like a "crop" or "zoom" defined in CSS, independent of `object-fit`.

A shorthand for positioning (`top`, `right`, `bottom`, `left`) absolutely positioned elements relative to their container.

14. Text Overflow

Manage how text is displayed when it overflows its container, commonly using `ellipsis` (...).

15. Comparison Functions (clamp(), min(), max())

Define dynamic CSS values that clamp between a minimum and maximum, or choose the smaller/larger of multiple values.

My width = clamp(150px, 50%, 400px). Resize the browser to see it change between these bounds.
